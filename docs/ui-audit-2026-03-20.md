# BreakTheVibe UI Audit Report

**Date:** 2026-03-20
**Scope:** All 15 templates, CSS design system, static assets
**Design Principles:** Precise & Confident, Data over Decoration, Instant Clarity

---

## Anti-Patterns Verdict

**MIXED — Some AI slop, mostly solid foundation.**

**Passes:**
- No gradient text, no glassmorphism, no hero metrics section
- No generic stock illustrations or decorative SVGs
- Good font choices (Inter + JetBrains Mono) — not generic
- Color palette is cohesive, not the default AI rainbow
- Dark mode is properly implemented with CSS variables

**Fails (AI tells detected):**
- Card grid layout on projects page — the default AI dashboard pattern
- Empty states are generic (icon + "No X yet" + "Do Y to get started") — no personality
- Confirm dialog is a basic div overlay, not a `<dialog>` element
- Toast system uses basic slide-up pattern — functional but forgettable
- Inline scripts in every template instead of extracted JS modules
- Tailwind CDN instead of built Tailwind — development-only pattern shipping to production

## Critical Issues

### CRIT-1: Toast container missing `aria-live`
**File:** `base.html:106`
**Category:** Accessibility (WCAG 4.1.3)
**Impact:** Screen reader users never hear success/error notifications. They have no idea if their action worked.
**Fix:** Add `aria-live="polite"` and `role="status"` to `#toast-container`
**Command:** `/impeccable:harden`

### CRIT-2: Confirm dialog uses `<div>` instead of `<dialog>`
**File:** `base.html:109-121`
**Category:** Accessibility (WCAG 2.1.2)
**Impact:** No focus trapping, no Escape key handling, background content still interactive for keyboard/screen reader users. Can tab behind the overlay.
**Fix:** Replace with native `<dialog>` element, use `showModal()` API
**Command:** `/impeccable:harden`

### CRIT-3: Pipeline buttons have no disabled state during execution
**File:** `project_detail.html` (btn-crawl, btn-generate, btn-run)
**Category:** UX / Data Integrity
**Impact:** User can trigger multiple concurrent pipelines by clicking rapidly. `hx-disabled-elt` is present but the visual feedback is unclear — no spinner, no "Running..." label change.
**Fix:** Add loading spinner SVG + text change ("Crawl" → "Crawling...") + `pointer-events: none` during active SSE
**Command:** `/impeccable:animate`

### CRIT-4: No focus management on page transitions
**File:** All templates
**Category:** Accessibility (WCAG 2.4.3)
**Impact:** After HTMX partial updates or page loads, focus is lost. Keyboard users are stranded at the top of the page.
**Fix:** Set focus to the main heading or the updated content region after navigation
**Command:** `/impeccable:harden`

## High-Severity Issues

### HIGH-1: Login/register pages have different layout than main app
**Files:** `login.html`, `register.html` extend `base_auth.html`
**Category:** Consistency
**Impact:** Jarring visual transition between auth pages and dashboard. Different nav, different spacing, feels like two different apps.
**Fix:** Unify auth layout — centered card over the same background, with simplified nav
**Command:** `/impeccable:normalize`

### HIGH-2: Form inputs missing `aria-describedby` for help text
**Files:** `llm_settings.html`, `rules_editor.html`, `login.html`
**Category:** Accessibility (WCAG 1.3.1)
**Impact:** Help text near inputs is visually associated but not programmatically linked. Screen readers skip it.
**Fix:** Add `id` to help text elements, link with `aria-describedby` on inputs
**Command:** `/impeccable:harden`

### HIGH-3: No `aria-busy` or progress announcements during SSE
**File:** `project_detail.html` (pipeline progress)
**Category:** Accessibility (WCAG 4.1.3)
**Impact:** Screen reader users get no feedback during crawl/generate/run. They don't know the pipeline is active.
**Fix:** Add `aria-busy="true"` to progress region, use `aria-live="polite"` for stage updates
**Command:** `/impeccable:harden`

### HIGH-4: Error display in login uses inline styles
**File:** `login.html`
**Category:** Theming / Consistency
**Impact:** Hardcoded `style="background: var(--callout-destructive-bg); ..."` bypasses the `.callout-destructive` class that already exists in the design system.
**Fix:** Replace inline styles with `.callout.callout-destructive` class
**Command:** `/impeccable:normalize`

### HIGH-5: Tab system doesn't handle keyboard navigation
**File:** `project_detail.html`, `test_result_detail.html`
**Category:** Accessibility (WCAG 2.1.1)
**Impact:** Arrow keys don't move between tabs. Only Tab key works, which is incorrect WAI-ARIA tab pattern. Left/Right arrow should switch tabs.
**Fix:** Add keydown handler for ArrowLeft/ArrowRight on `role="tablist"`
**Command:** `/impeccable:harden`

### HIGH-6: Empty states lack personality and actionability
**Files:** `projects.html:50-57`, `test_runs.html:44-51`
**Category:** UX / Onboarding
**Impact:** Generic "No X yet. Do Y." tells users nothing useful. No guidance on what to expect, no call-to-action button.
**Fix:** Add contextual guidance, inline action buttons, and brief explanation of what happens next
**Command:** `/impeccable:onboard`

### HIGH-7: Tailwind CDN in production
**File:** `base.html:25`
**Category:** Performance
**Impact:** Loading full Tailwind (~300KB) from CDN on every page load. Blocks rendering. Should be purged and bundled.
**Fix:** Build Tailwind with purge, inline critical CSS, load rest async
**Command:** `/impeccable:optimize`

### HIGH-8: All inline scripts are duplicated per template
**Files:** Every template's `{% block scripts %}`
**Category:** Maintainability / Performance
**Impact:** Same patterns (fetch + error handling, state transitions, WebAuthn flows) are copy-pasted across templates. Changes require updating multiple files.
**Fix:** Extract shared JS to `static/js/` modules (e.g., `pipeline.js`, `auth.js`, `forms.js`)
**Command:** `/impeccable:extract`

## Medium-Severity Issues

### MED-1: Delete button hover goes straight to destructive red
**File:** `projects.html:79`
**Impact:** No intermediate state — feels aggressive
**Fix:** Add subtle bg tint on hover before the red color change
**Command:** `/impeccable:animate`

### MED-2: No skeleton/loading state for initial page content
**Files:** All list pages
**Impact:** Content pops in after load, no smooth transition
**Fix:** Add CSS skeleton placeholders or fade-in on content load
**Command:** `/impeccable:animate`

### MED-3: Rules editor textarea has no line numbers
**File:** `rules_editor.html`
**Impact:** YAML editing without line numbers makes debugging errors harder
**Fix:** Add a line-number gutter (CSS counter or JS-based)
**Command:** `/impeccable:delight`

### MED-4: Video player has no custom controls styling
**File:** `test_result_detail.html`
**Impact:** Browser default video controls look inconsistent with the design system
**Fix:** Style or hide default controls, add custom play/pause
**Command:** `/impeccable:normalize`

### MED-5: Pagination doesn't preserve view mode (grid/list)
**File:** `projects.html`
**Impact:** Clicking a pagination link reloads the page, resetting to grid view (localStorage restores it, but there's a flash)
**Fix:** Add view parameter to pagination URLs or use HTMX for pagination
**Command:** `/impeccable:harden`

### MED-6: Toast close button uses text `x` instead of icon
**File:** `base.html:150`
**Impact:** Inconsistent with the rest of the UI which uses SVG icons
**Fix:** Replace with an SVG close icon matching the design system
**Command:** `/impeccable:normalize`

### MED-7: Stat cards in test results have no trend indicator
**File:** `test_result_detail.html`
**Impact:** Shows totals but no context — is this better or worse than last run?
**Fix:** Add comparison to previous run (e.g., "+2 passed", "-1 failed")
**Command:** `/impeccable:delight`

### MED-8: Settings save button has state transitions but no error state
**File:** `llm_settings.html`
**Impact:** If save fails, button returns to "Save" with no indication of failure
**Fix:** Add error state: "Failed" with destructive styling, then reset
**Command:** `/impeccable:harden`

### MED-9: Project status badge text is raw enum value
**Files:** `projects.html:74`, `project_detail.html`
**Impact:** Shows "created", "running" — could be more human: "Ready", "In Progress"
**Fix:** Map status values to human-readable labels
**Command:** `/impeccable:clarify`

### MED-10: Sitemap page loads D3.js even when mindmap tab isn't active
**File:** `sitemap.html`
**Impact:** Unnecessary JS execution, potential layout shift
**Fix:** Lazy-load D3.js only when mindmap tab is activated
**Command:** `/impeccable:optimize`

### MED-11: No `prefers-reduced-motion` media query
**File:** `style.css`
**Impact:** Users who prefer reduced motion still see all animations
**Fix:** Add `@media (prefers-reduced-motion: reduce)` to disable/simplify animations
**Command:** `/impeccable:harden`

### MED-12: Select dropdowns have no custom styling
**Files:** `llm_settings.html`
**Impact:** Native `<select>` elements look inconsistent across browsers and break dark mode on some platforms
**Fix:** Add `.select` class styling in design system
**Command:** `/impeccable:normalize`

## Low-Severity Issues

### LOW-1: Favicon is SVG — no PNG fallback
**File:** `base.html:7`
**Impact:** Older browsers may not display favicon
**Fix:** Add `<link rel="icon" type="image/png" href="...">` fallback

### LOW-2: Google Fonts loaded without `font-display: swap`
**File:** `base.html:22`
**Impact:** Potential FOIT (Flash of Invisible Text) on slow connections
**Fix:** Append `&display=swap` to Google Fonts URL

### LOW-3: `stagger-children` class referenced but not defined in CSS
**Files:** `projects.html:61`, `test_runs.html:14`
**Impact:** No stagger animation actually occurs — dead CSS class
**Fix:** Either add the animation or remove the class

### LOW-4: `animate-in` class on new-project-form has no definition
**File:** `projects.html:25`
**Impact:** No animation when form appears
**Fix:** Define or remove

### LOW-5: Some buttons missing explicit `type="button"`
**Files:** Various
**Impact:** Buttons inside forms default to `type="submit"`, may cause unexpected submissions
**Fix:** Add `type="button"` to non-submit buttons inside forms

### LOW-6: Accordion chevron animation could be smoother
**File:** `style.css` (`.accordion summary::after`)
**Impact:** Rotation is functional but abrupt
**Fix:** Add transition to the `::after` rotate transform

## Positive Findings

1. **Excellent token system** — 61 CSS variables covering every semantic color, all with dark mode variants
2. **Clean component naming** — `.btn-primary`, `.badge-success`, `.card-interactive` — consistent BEM-like convention
3. **Good ARIA on tabs** — `role="tablist"`, `role="tab"`, `aria-selected` properly implemented
4. **Dark mode is comprehensive** — Every component responds to `.dark` class, no broken contrast
5. **HTMX usage is clean** — Progressive enhancement, no over-engineering
6. **Responsive grid system** — Consistent `grid-cols-1 sm:grid-cols-2 lg:grid-cols-3` pattern
7. **Toast system** — Well-implemented with dismiss, auto-close, and visual hierarchy by type
